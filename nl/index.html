ProxmoxVue

Native iPad-app voor het beheren van je Proxmox VE-cluster. Touch-first, direct by default.

App Store Binnenkort iPadOS 17+
// waarom

Gemaakt voor de iPad.

iPad-first

Drie-koloms split view, werkt in portrait én landscape. Grote tap-targets, long-press menu's — geen hover-only UI overgewaaid uit een phone-layout.

Native SwiftUI

Geen Electron, geen web-view wrapper. Puur SwiftUI + SwiftData. Start instant, respecteert system themes, koude start in <300 ms.

Direct API first

Je iPad praat rechtstreeks met je Proxmox-server. Je credentials blijven in de iOS Keychain. Optionele cloud-functies — push-notificaties, remote access en een cross-cluster view — draaien op een self-hosted daemon die jij beheert.

// zo werkt het

Drie stappen van nul naar cluster.

01

Voeg je host toe

Vul je Proxmox-URL in (pve.example.com of IP+poort), zet de TLS-toggle aan als je een self-signed cert hebt, en kies hoe je wil inloggen.

02

Zie je cluster

Live dashboard: nodes met CPU/RAM/opslag, VM- en LXC-lijsten per node, realtime status. Actuele waarden, geen verouderde polls.

03

Onderneem actie

Start / stop / reboot / shutdown van VM's en containers. Zie taken aflopen in de log-viewer. Alles wordt achter de schermen gesigneerd met CSRFPreventionToken.

// authenticatie

Log in op de manier die je al gebruikt.

/access/ticket

Gebruikersnaam + wachtwoord

Dezelfde realm-picker als de Proxmox web-UI. PAM, PVE, LDAP of AD. Tickets worden automatisch vernieuwd voor ze verlopen.

/access/openid

OIDC — Authentik, Keycloak

Volledige browser-gebaseerde OpenID Connect-flow via ASWebAuthenticationSession. Je SSO blijft bij je eigen provider.

PVEAPIToken

API Token

De klassieke scoped tokens voor power-users en automation. Opgeslagen in de iOS Keychain, nooit gesynct.

Geen account nodig in de standaardmodus.

In directe modus blijft alles tussen je iPad en je Proxmox-server — je credentials, je hostnames, je VM's. Cloud-functies zijn opt-in en strak afgebakend. Lees het privacybeleid.
