Veelgestelde vragen

Laatst bijgewerkt: 21 april 2026

Welke Proxmox-versies worden ondersteund?

Proxmox VE 7.x en 8.x zijn getest. De app gebruikt standaard REST-endpoints (/api2/json/...) die al jaren stabiel zijn. Als iets stuk gaat op een specifieke minor release, meld het via support.

Werkt het met een cluster of alleen één node?

Allebei. De app bevraagt /cluster/resources, dat retourneert alle nodes, VM's, containers en storages. Single-node installaties zien er hetzelfde uit als een één-node cluster.

Heb ik een backend, relay of account nodig?

Voor standaard direct-API-gebruik: nee. Je iPad praat direct met je Proxmox-server — geen account bij ons, geen cloud-relay. Wil je de optionele cloud-functies (push, remote access, cross-cluster view), dan installeer je een kleine self-hosted daemon op een machine in je netwerk en neem je een Cloud-abonnement. De daemon praat met onze EU-hosted Supabase-backend, geeft je Proxmox-credentials nooit door, en je kunt zijn uitgaande verkeer in je firewall beperken tot één endpoint.

Wat zijn cloud-functies, en hoe werken ze?

Cloud-functies zijn opt-in add-ons: push-notificaties (node offline, backup gefaald, VM down), remote access als je buiten je lokale netwerk zit, en een cross-cluster aggregated dashboard over meerdere Proxmox-hosts.

Ze vereisen proxmoxvue-agent, een kleine self-hosted daemon. Je draait hem op de Proxmox-host zelf, op een willekeurige Linux-machine in hetzelfde netwerk (een Raspberry Pi 4/5 met 64-bit OS werkt prima), of als Docker-container — wat het beste bij je setup past. Eenmalig installeren en de app pakt het cloud-pad automatisch op voor geabonneerde users. Je iPad blijft direct-API gebruiken voor al het andere.

Kan ik de agent op een aparte machine, een Raspberry Pi of in Docker draaien?

Ja — alle drie. De agent is een enkele Go-binary met drie deployment-opties:

  • Op je Proxmox-host — simpelste optie, minste bewegende delen. De standaardinstall gebruikt https://127.0.0.1:8006 voor de Proxmox-API.
  • Op een aparte Linux-machine in je netwerk — houdt je hypervisor schoon. We leveren amd64- en arm64-binaries, dus een Raspberry Pi 4 of 5 met 64-bit Raspberry Pi OS of Ubuntu werkt direct. Oudere Pi 3 / 32-bit ARM (armv7) wordt niet ondersteund door de meegeleverde binary. De agent bereikt je Proxmox-host over je LAN op poort 8006.
  • Als Docker-container — zelfde binary, geconfigureerd via environment variables. Zie DOCKER.md voor de env-var reference en een docker-compose.yml-voorbeeld.

Eén agent-enrollment is gekoppeld aan één Proxmox-host — voor meerdere hosts draai je meerdere instances (aparte machines, of één machine met meerdere containers).

Heeft de daemon firewall-wijzigingen nodig?

Geen inkomende firewall-regels. De daemon maakt alleen uitgaande verbindingen — naar één endpoint (onze EU-hosted Supabase-backend).

Je kunt het uitgaande verkeer optioneel beperken in je eigen firewall tot alleen die ene host: volledige controle over waar de daemon naartoe praat. Omdat verbindingen door de daemon worden opgezet hoeft er geen poort op je router open.

Waar haal ik de agent vandaan, en hoe installeer ik hem?

proxmoxvue-agent is open source. De repository — inclusief build-instructies, een voorbeeld systemd-unit en config-templates — staat op GitHub: github.com/TheLion/proxmoxvue-agent.

Je hebt hem alleen nodig als je de cloud-functies wil (push, remote access, cross-cluster view). Voor standaard direct-API-gebruik van de app is de agent niet nodig.

Hoe log ik in?

Drie manieren, kies wat bij je setup past:

Gebruikersnaam + wachtwoord + realm — hetzelfde als de Proxmox web-UI. PAM, PVE, LDAP of AD.

OIDC — als je realm Authentik, Keycloak of een andere OpenID Connect-provider is, opent de app je browser voor de login en vangt de callback op. Je moet proxmoxvue-auth://callback als redirect-URI toestaan in je provider.

API Token — klassieke scoped tokens. Toggle "Gebruik API-token" in het add-host scherm.

Mijn Proxmox gebruikt een self-signed certificaat. Werkt dat?

Ja. Zet de toggle "Self-signed TLS accepteren" aan bij het toevoegen van de host. De app schakelt certificaatvalidatie uit alleen voor die host — andere hosts blijven normaal gevalideerd.

Waar worden mijn credentials opgeslagen?

In de iOS Keychain. Dat betekent:

— versleuteld opgeslagen door het OS,
— alleen toegankelijk als het apparaat ontgrendeld is,
— niet gesynct naar iCloud,
— beperkt tot deze app.

OIDC-sessie-tickets leven alleen in het geheugen — nooit gepersisteerd. Een nieuwe browser-login is nodig bij expiry.

Waarom iPad-first?

Op iPhone bestaan al meerdere Proxmox-apps. Op iPad niet. Die leegte vullen we met tab-navigatie over cluster/nodes/VMs/LXC/storage met drill-down detail-views, long-press menu's, grote tap-targets en een distinctief donker terminal-aesthetic — een touch-native tool ontworpen voor het formaat, niet een phone-UI uitgerekt naar 11 inch. Een iPhone-companion staat op de roadmap (push-notificaties, widgets, quick actions) maar wordt geen volledige management-tool.

Komt er een Mac-versie?

Nee. De Proxmox web-UI op desktop is goed genoeg; een Mac-app zou te weinig toevoegen om de bouwlast te rechtvaardigen.

Is de code open source?

De iPad-app zelf is closed source — we overwegen het later eventueel opnieuw.

De cloud-agent (proxmoxvue-agent) is wél open source: github.com/TheLion/proxmoxvue-agent. Je draait hem zelf en kunt exact zien wat hij verstuurt.
